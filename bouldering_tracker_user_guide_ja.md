# ボルダリングトラッカー — 使い方ガイド

## このスプレッドシートでできること

このスプレッドシートは、クライマー、課題、トライ履歴、完登状況、ポイント、プロフィール画面を管理するためのものです。

基本の流れ：

1. **Customers** にクライマーを追加・編集する。
2. **Routes** に課題を追加・編集する。
3. **Tracker Tools → Sync IDs & Dashboards** を実行する。
4. **Data** または **Customer Profile** から登った記録を追加する。
5. チェックボックスで顧客/課題プロフィールへ移動する。
6. 顧客向け表示タブ（**Rankings View** / **New Routes**）を更新する。

---

## 初回セットアップ

事前条件（Googleアカウントごとに一度だけ）：
- Apps Script API アクセスを有効化してください：`https://script.google.com/home/usersettings`
- 有効化直後は、セットアップ/プッシュ前に 1〜3 分待ってください。

Apps Script を貼り付け・更新した後：

1. スプレッドシートを再読み込みします。
2. カスタムメニュー **🧗‍♂️ Tracker Tools** を開きます。
3. **Setup / Repair Spreadsheet** をクリックします。
4. Google の権限承認画面が出たら承認します。
5. 必要に応じて **Sync IDs & Dashboards** を一度実行します。

**Install / Repair Edit Trigger** は、以下の場合に再実行してください：

- スクリプトを初めて追加したとき
- スプレッドシートをコピーしたとき
- チェックボックスで画面移動できなくなったとき
- トリガーが削除・破損したとき

## シート一覧

### Data

メインのダッシュボードです。

主な項目：

- **B1 / C1** — 開始日・開始時間フィルター。ここを編集するとダッシュボードが更新されます。
- ペインは 3 行目 / C 列まで固定されます。
- **B2** — 登った記録を追加する顧客を選択します。
- **B3** — 登った課題を選択します。
- **C3** — ステータスを選択します。
- **A列のチェックボックス** — 顧客プロフィールを開きます。
- **1行目の課題チェックボックス** — 課題プロフィールを開きます。
- **4行目以降 / D列以降** — 顧客ごとの各課題のベストステータスを表示します。

### Settings

ステータス記号と点数設定を管理します。

デフォルト設定：

- **◎** — Flash（一撃）— 10点 — 完登扱い
- **◯** — Red-Point（完登）— 7点 — 完登扱い
- **△** — Foot-Follow — 5点 — 完登扱いではない
- **✓** — Attempt（トライ）— 1点 — 完登扱いではない

**Complete?** 列で、そのステータスを完登として扱うかどうかを設定します。

### Routes

課題データベースです。

列：

- **ID** — 自動生成される課題ID。例：`R-500102`
- **Name** — 課題名
- **V Scale Difficulty** — Vグレードの数字。例：V4なら `4`
- **Link** — 課題の画像・動画・参考URLなど

課題名は重複しても問題ありません。実際の識別には **Route ID** を使います。

### Customers

顧客/クライマーデータベースです。

列：

- **ID** — 自動生成される顧客ID。例：`C-04E020`
- **Name** — 顧客名
- **Completion Rate** — 自動計算される完登率
- **Points** — 自動計算されるポイント
- **V Scale Level** — 完登した課題から計算されるレベル
- **Birthday** — 任意
- **Age** — 自動計算
- **Height** — 任意
- **Experience** — 任意
- **Gender** — 任意

### Customer Profile

特定の顧客のプロフィール画面です。

- **A1** に現在表示している顧客IDが入ります。
- 右側のクイック入力欄：
  - **J2** ルート
  - **J3** ステータス
  - **K3** チェックで登攀イベント送信
- トレーニング入力欄：
  - **J8** 指標
  - **J9** 値
  - **J10** 単位（Settings の指標設定から自動入力）
  - **J11** メモ
  - **K11** チェックでトレーニングイベント送信

### Route Profile

特定の課題のプロフィール画面です。

- **A1** に現在表示している課題IDが入ります。

### Rankings View

顧客向けのランキング表示タブです。

- **Japanese Level** と **V Scale Level** の両方を表示します。
- シート上の `Sort By` で並び替え項目を選べます。
- `Customers` のデータから作成されます。

### New Routes

最近追加された課題を表示する顧客向けタブです。

- ルート追加日時（Routes の E列）を利用します。
- スクリプトの初期設定は「直近14日」です。
- 新しい課題ほど上に表示されます。

### EventLog

運用イベントログ（登攀ログ追加・トレーニング記録追加など）を保存します。監査や復旧時の確認に使います。

### Logbook

登った記録の元データです。

通常は手動編集しないでください。明確な修正が必要なときだけ編集します。

主な列：

- タイムスタンプ
- 顧客ID
- 課題ID
- ステータス
- グレード
- 身長、年齢、経験、性別などの記録時点の顧客情報

## 通常の使い方

### 新しい顧客を追加する

1. **Customers** を開きます。
2. **Name** 列の次の空行に顧客名を入力します。
3. 必要なら誕生日、身長、経験、性別を入力します。
4. **Tracker Tools → Sync IDs & Dashboards** を実行します。
5. 顧客IDが自動生成されます。

基本的にIDは手入力しないでください。

### 新しい課題を追加する

1. **Routes** を開きます。
2. **Name** 列の次の空行に課題名を入力します。
3. Vグレードの数字を入力します。
4. 必要ならリンクを入力します。
5. **Tracker Tools → Sync IDs & Dashboards** を実行します。
6. 課題IDが自動生成されます。

課題のドロップダウンは次の形式になります：

```text
課題名 (R-ABC123)
```

同じ名前の課題があっても、IDで区別できます。

### Data シートから登った記録を追加する

1. **Data** を開きます。
2. **B2** で顧客を選択します。
3. **B3** で課題を選択します。
4. **C3** でステータスを選択します。
5. Data 用の記録追加ボタン/関数がある場合、それを実行します。
6. 記録が **Logbook** に追加され、ダッシュボードが更新されます。

### Customer Profile から登った記録を追加する

1. **Data** のA列チェックボックスをクリックして顧客プロフィールを開きます。
2. **Customer Profile** で課題とステータスを選択します。
3. Profile 用の記録追加ボタン/関数がある場合、それを実行します。
4. 記録が **Logbook** に追加され、ダッシュボードが更新されます。

### 顧客プロフィールを開く

1. **Data** を開きます。
2. 対象顧客の左側、**A列** のチェックボックスをクリックします。
3. **Customer Profile** に移動します。

動かない場合は、以下を実行してください：

**Tracker Tools → Install / Repair Edit Trigger**

### 課題プロフィールを開く

1. **Data** を開きます。
2. 対象課題の上、**1行目** のチェックボックスをクリックします。
3. **Route Profile** に移動します。

動かない場合は、以下を実行してください：

**Tracker Tools → Install / Repair Edit Trigger**

### 顧客向け表示タブを更新する

以下を実行してください：

**Tracker Tools → Refresh Rankings & New Routes Views**

このメニューがない場合は、Apps Script プロジェクトに `bouldering_tracker_views.gs` を追加し、`onOpen()` にメニュー項目を追加してください。

### 手動イベント入力と移行

- **Prepare Event Entry Tab**: Customer / Route / Climb / Training のガイド付き入力ブロック＋上級者向け行入力を作成します。
- **Apply Event Entry Rows**: チェック済みの上級者向け行を `EventLog` に追加し、テーブルを再構築します。
- **Migrate Existing Tables to EventLog**: 既存の `Customers/Routes/Logbook/TrainingLog` を `EventLog` に移行します。
- **Run Post-Update Routine**: スクリプト更新後の推奨ワンクリック処理です。
- **Reset Data (Safe)**: 生成/集計タブをクリアし、必要に応じて EventLog も削除します。

## Sync IDs & Dashboards の役割

この機能は以下を行います：

1. 足りない顧客ID・課題IDを自動生成する。
2. **Customers** の計算式を更新する。
3. **Data** ダッシュボードを再構築する。
4. **Data!B1:C1** の日付/時間フィルターを適用する。
5. **Logbook** を読み取り、各顧客×各課題のベストステータスを表示する。
6. 顧客、課題、ステータスのドロップダウンを更新する。

顧客、課題、設定を変更した後や、ダッシュボードが古いと感じたときに実行してください。

顧客向け表示を更新したい場合は、その後に **Refresh Rankings & New Routes Views** も実行してください。

---

## Install / Repair Edit Trigger の役割

このメニューは、`handleEdit(e)` というスクリプト関数に対して **インストール型編集トリガー** を設定します。

これにより、以下の動作が安定して動きます：

- **Data!B1** または **Data!C1** を編集するとダッシュボードを更新する。
- 顧客チェックボックスをクリックすると **Customer Profile** を開く。
- 課題チェックボックスをクリックすると **Route Profile** を開く。

また、古い重複した `handleEdit` トリガーがあれば削除してから、新しいトリガーを作成します。

重要：この機能自体は登った記録を追加しません。編集操作とナビゲーション/フィルター更新スクリプトを接続するためのものです。

## トラブルシューティング

### チェックボックスを押しても移動しない

以下を実行してください：

**Tracker Tools → Install / Repair Edit Trigger**

Google から承認を求められたら承認してください。

### 新しい顧客や課題が表示されない

以下を実行してください：

**Tracker Tools → Sync IDs & Dashboards**

### 課題名が重複していて分かりにくい

課題名が同じでも問題ありません。正しい課題IDを含む項目を選んでください：

```text
課題名 (R-ABC123)
```

### ダッシュボードの数字が古い

以下を実行してください：

**Tracker Tools → Sync IDs & Dashboards**

### Logbook の内容がおかしい

各行に以下が入っているか確認してください：

- 正しいタイムスタンプ
- 顧客ID
- 課題ID
- ステータス記号

明確な修正が必要な場合以外、Logbook は手動編集しないでください。
