# Gemini / AI Spreadsheet Assistant Guide (for this tracker)

Use this guide when asking Google built-in AI tools (e.g., Gemini in Sheets) to modify or analyze the bouldering tracker.

## 1) Project context to provide first
Paste this context before asking for changes:

- This workbook is an **event-driven bouldering tracker**.
- Source-of-truth tabs: `Customers`, `Routes`, `Logbook`, `TrainingLog`, `Settings`, `EventLog`.
- Generated tabs: `Data`, `Customer Profile`, `Route Profile`, `Rankings View`, `New Routes`.
- IDs are auto-generated by script (`C-...`, `R-...`) and should be treated as **system/calculated**.
- Status scoring: `◎=10`, `◯=7`, `△=5`, `✓=1`.
- Completion-rate logic for routes uses only `◎` and `◯` as completes.

## 2) Safety constraints (important)
Ask Gemini to follow these constraints:

1. Do not rename source-of-truth tabs.
2. Do not delete script-linked columns unless explicitly requested.
3. Do not overwrite formula columns in bulk without backup.
4. Keep bilingual labels consistent (EN/JA where applicable).
5. For structural changes, suggest steps first, then apply after confirmation.

## 3) Prompt template (copy/paste)

"You are editing a Google Sheet bouldering tracker.
Follow these constraints:
- Keep source tabs intact: Customers, Routes, Logbook, TrainingLog, Settings, EventLog.
- IDs are auto-generated and should be treated as calculated.
- Preserve formula columns and data validations unless I explicitly ask to replace them.
- Keep UI labels consistent with current language mode.

Task:
[describe exact change]

Return:
1) cells/ranges you will change,
2) formulas/validation changes,
3) risks,
4) rollback steps."

## 4) Good requests examples

- "Add optional member fields to Customers after existing profile fields, without changing current formulas in C, D, E, G, K, L."
- "Color-code header rows by required/optional/calculated, but do not change data rows."
- "Suggest a validation rule update for Event Entry input cells only."

## 5) Verification checklist after AI edits

- Run `Tracker Tools -> Sync IDs & Dashboards`
- Run `Tracker Tools -> Run Post-Update Routine`
- Check:
  - `Data` quick action checkbox still works (`D3`)
  - Customers/Routes header colors are still correct
  - Event Entry single-entry colors and validations are intact
  - Rankings and profile tabs still refresh

## 6) Common migration scenario: importing existing users in bulk

If the gym already has a small member table and wants to import users quickly:

Recommended path:
1. Keep a backup copy of the sheet first.
2. Ask Gemini to generate a one-time Apps Script import helper.
3. Import into `Customers` while preserving existing computed columns/formulas.
4. Run:
   - `Tracker Tools -> Sync IDs & Dashboards`
   - `Tracker Tools -> Run Post-Update Routine`

Prompt template for this case:

"Create a one-time Google Apps Script function to bulk import users from sheet `RawUsers` into `Customers`.
Constraints:
- Do not overwrite formula columns (C, D, E, G, K, L).
- If ID is missing, leave it blank so tracker auto-ID logic can fill it.
- Map fields safely by header names, not hard-coded indexes.
- Skip fully empty rows.
- Log how many rows were inserted/updated/skipped.
- Return rollback steps."

Tip: if source headers are messy, ask Gemini to output a header mapping table first and wait for confirmation before writing data.

## 7) If something breaks

1. Use Undo immediately if possible.
2. Re-run:
   - `Tracker Tools -> Rebuild Tables from EventLog`
   - `Tracker Tools -> Run Post-Update Routine`
3. Capture screenshot evidence (tab name + cell refs + menu action).
4. Report via GitHub issue template.
